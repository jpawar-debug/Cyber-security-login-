# 🛡️ CYBER SECURITY ACCESS CONTROL SYSTEM

A SECURE COMMAND-LINE INTERFACE ACCESS CONTROL SYSTEM DEVELOPED IN C PROGRAMMING. THIS PROJECT SIMULATES REAL-WORLD CYBER SECURITY LOGIN PROTOCOLS AND showcase TWO ENHANCED VERSIONS BASED ON MEMORY ROUTING.

---

## 📌 PROJECT OVERVIEW

THIS SYSTEM PROTECTS SENSITIVE TERMINALS BY VALIDATING USER CREDENTIALS. IT USES ADVANCED STRING MANIPULATION TECHNIQUES TO ENSURE ONLY AUTHORIZED PERSONNEL GAIN ACCESS, ENFORCING AUTOMATIC MITIGATION PATTERNS UPON AUTHENTICATION FAILURE.

### 📂 SYSTEM ARCHITECTURE (TWO INTERACTIVE MODES)

#### 1. Static Authentication Engine (`fixed_auth_guard.c`)
* **PREDEFINED CREDENTIALS:** SYSTEM VALIDATES INPUT AGAINST ENCRYPTED/HARDCODED ADMINISTRATOR DATA (`jay_pawar`).
* **BUFFER VERIFICATION:** DEMONSTRATES FAST BUFFER COMPARISON AT COMPILE-TIME.

#### 2. Dynamic Registration Engine (`dynamic_registration_guard.c`)
* **RUNTIME REGISTRATION:** ALLOWS OPERATORS TO INTERACTIVELY REGISTER A NEW USERNAME AND SECRET PASSWORD FIRST.
* **MEMORY BUFFER COPYING:** LOGS RUNTIME INPUT INTO MAIN STORAGE REGISTERS DYNAMICALLY BEFORE EXECUTION.

---

## 🚀 KEY FEATURES

* **SECURE CREDENTIAL VALIDATION:** VERIFIES AUTHENTIC USERNAMES AND SECRET PASSWORDS USING NATIVE STRING ROUTINES.
* **LIMITED LOGIN ATTEMPTS:** USERS GET A MAXIMUM OF 3 CHANCES TO ENTER CORRECT DETAILS TO PREVENT BRUTE FORCE ATTEMPTS.
* **AUTOMATIC SYSTEM LOCKOUT:** INTENSIFIES SECURITY BY LOCKING THE TERMINAL AFTER 3 WRONG ATTEMPTS.
* **REAL-TIME STATUS WARNINGS:** ALERTS THE USER WITH REMAINING ATTEMPTS AFTER EVERY AUTHENTICATION FAILURE.

---

## 🛠️ CORE CONCEPTS USED

1. **FLAG-BASED WHILE LOOPS:** TO MANAGE, TRACK, AND TERMINATE AUTHENTICATION STATES CONTINUOUSLY.
2. **NESTED IF-ELSE STATEMENTS:** FOR CONDITIONAL CONDITIONAL DECISION-MAKING DURING REPEATED FAILURE DETECTION.
3. **STRING COPYING (STRCPY):** ROUTING RUNTIME USER SIGN-UP INPUT DIRECTLY INTO MASTER BUFFERS.
4. **STRING COMPARISON (STRCMP):** RECOGNIZING AUTHENTIC INPUT VECTORS BY RETURNING ZERO STATE SYNC.
