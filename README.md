# CYBER SECURITY LOGIN SYSTEM

A SECURE COMMAND-LINE INTERFACE ACCESS CONTROL SYSTEM DEVELOPED IN C PROGRAMMING. THIS PROJECT SIMULATES A REAL-WORLD CYBER SECURITY LOGIN PROTOCOL WITH LIMITED ATTEMPTS AND AUTOMATIC SYSTEM LOCKOUT MECHANISM.

---

## PROJECT OVERVIEW

THIS SYSTEM PROTECTS SENSITIVE TERMINALS BY VALIDATING USER CREDENTIALS. IT USES ADVANCED STRING COMPARISON TECHNIQUES TO ENSURE ONLY AUTHORIZED PERSONNEL GAIN ACCESS TO THE SYSTEM.

### KEY FEATURES
* **SECURE CREDENTIAL VALIDATION:** VERIFIES AUTHENTIC USERNAMES AND SECRET PASSWORDS.
* **LIMITED LOGIN ATTEMPTS:** USERS GET A MAXIMUM OF 3 CHANCES TO ENTER CORRECT DETAILS.
* **AUTOMATIC SYSTEM LOCKOUT:** INTENSIFIES SECURITY BY LOCKING THE TERMINAL AFTER 3 WRONG ATTEMPTS.
* **REAL-TIME STATUS WARNINGS:** ALERTS THE USER WITH REMAINING ATTEMPTS AFTER EVERY FAILURE.

---

## CORE CONCEPTS USED

1. **WHILE LOOPS:** TO MANAGE AND TRACK LOGIN ATTEMPTS CONTINUOUSLY.
2. **IF-ELSE STATEMENTS:** FOR CONDITIONAL DECISION-MAKING DURING AUTHENTICATION.
3. **CHARACTER ARRAYS (STRINGS):** TO STORE SECURE USER INPUTS.
4. **STRING COMPARISON (STRCMP):** CORE LOGIC USED TO COMPARE ENTERED CREDENTIALS WITH STORED DATA.
